Teoría: Entradas digitales 20 min
ElectrónicaProgramación
Un pin digital puede ser ENTRADA. Leerás HIGH (1) o LOW (0). Con un pulsador, conectamos una resistencia pull-down para que cuando no se presione, el pin lea LOW (0).
🔧 Simular en Tinkercad
❓ ¿Qué es pull-down? Una resistencia que asegura que el pin esté en 0V cuando el botón está abierto.
Práctica 6: Pulsador y LED 55 min
ProgramaciónElectrónica
Conecta un pulsador al pin 15 (con resistencia pull-down de 10kΩ) y un LED al pin 4. Programa: mientras presiones el botón, el LED se enciende.
int boton = 15; int led = 4; void setup() { pinMode(boton, INPUT); pinMode(led, OUTPUT); Serial.begin(115200); } void loop() { int estado = digitalRead(boton); Serial.println(estado); if (estado == HIGH) { digitalWrite(led, HIGH); } else { digitalWrite(led, LOW); } }
💡 TIP: Abre el Monitor Serie (115200 baudios) y verás los 1s y 0s cuando presionas.

Experimentación: Variaciones con pulsador 30 min
Ahora haz que el LED se encienda y apague con cada pulsación (modo toggle). Pista: necesitas recordar el estado anterior. Y cuenta cuántas veces presionas, muéstralo en el Monitor Serie.
// Ejemplo de toggle (estructura básica) int estadoBoton; int ledEncendido = false; int ultimoEstado = LOW; void loop() { estadoBoton = digitalRead(boton); if (estadoBoton == HIGH && ultimoEstado == LOW) { ledEncendido = !ledEncendido; digitalWrite(led, ledEncendido); delay(50); // antirrebote simple } ultimoEstado = estadoBoton; }

DÍA 5 · 2.5 hrs Entradas Analógicas (LDR y Potenciómetro)
⏱️ 2.5 hrs
Repaso y tarea15 min
¿Qué encontraste sobre señal analógica? En la naturaleza, casi todo es analógico: la luz, la temperatura, la posición de una perilla.
Teoría: Entradas analógicas (ADC)20 min
El ESP32 lee voltajes entre 0 y 3.3V y los convierte a un número entre 0 y 4095. Eso es un ADC (Convertidor Analógico-Digital).
⚡ Simulador ADC
Práctica 7: LDR (Luz Nocturna)40 min
Conecta LDR con un divisor de tensión (LDR + resistencia 10kΩ) al pin 34. Programa: si el valor es menor a 2000 (oscuridad), enciende un LED.
int ldrPin = 34; int led = 4; int valor; void setup() { Serial.begin(115200); pinMode(led, OUTPUT); } void loop() { valor = analogRead(ldrPin); Serial.println(valor); if (valor < 2000) { digitalWrite(led, HIGH); // oscuro -> enciende } else { digitalWrite(led, LOW); } delay(500); }
💡 TIP: Ajusta el umbral 2000 según tu luz ambiente. Cúbrelo con la mano para probar.
🌐 Simulación LDR
☕ Receso15 min
Práctica 8: Potenciómetro (control de brillo)35 min
Conecta potenciómetro: patas extremas a 3.3V y GND, la central al pin 35. Usa `analogRead` y convierte a PWM con `map` para controlar brillo de un LED.
int potPin = 35; int ledPin = 4; int potValor, brillo; void loop() { potValor = analogRead(potPin); brillo = map(potValor, 0, 4095, 0, 255); analogWrite(ledPin, brillo); delay(50); }

DÍA 6 · 2.5 hrs Sensor PIR y Evaluación de Mitad de Curso
⏱️ 2.5 hrs
Teoría: Sensor PIR20 min
PIR detecta movimiento por infrarrojos. Tiene tres pines: VCC (5V), GND, y señal (OUT). Al detectar movimiento, la señal se pone HIGH.
📘 Guía PIR
Práctica 9: Alarma de movimiento40 min
Conecta PIR al pin 14, LED al 4 y buzzer al 5. Programa: cuando el PIR detecte movimiento, enciende LED y activa buzzer 2 segundos.
int pirPin = 14; int led = 4; int buzzer = 5; void setup() { pinMode(pirPin, INPUT); pinMode(led, OUTPUT); pinMode(buzzer, OUTPUT); Serial.begin(115200); } void loop() { if (digitalRead(pirPin) == HIGH) { Serial.println("Movimiento detectado"); digitalWrite(led, HIGH); tone(buzzer, 1000, 2000); delay(2000); digitalWrite(led, LOW); } }

Evaluación Parcial: Proyecto "Habitación Inteligente"40 min
Cargando..., este es tu reto: Construye un sistema que:
  • Encienda un LED si el PIR detecta movimiento Y la LDR detecta oscuridad (noche).
  • Muestre en el Monitor Serie el estado de ambos sensores.
  • Use un pulsador para activar/desactivar todo el sistema (enable/disable).

❓ Preguntas que se hizo Cargando... esta semana

¿Por qué mi botón no funciona bien? Probablemente falta resistencia pull-down o el código no maneja rebotes. Revisa conexiones y agrega un pequeño delay.
Mi LDR siempre da valores extremos Revisa el divisor de tensión. Usa resistencias de 10kΩ. Conecta bien los cables.
El PIR se activa solo todo el tiempo Ajusta los potenciómetros del sensor (sensibilidad y tiempo). Aléjalo de corrientes de aire.

🧭 Notas del guía (soporte)

Historia para mantener: "Cargando..., en una fábrica automatizada, los sensores como el PIR detectan presencia, los LDR controlan luces, los botones son interfaces. Estás aprendiendo justo lo que se usa en la industria."

Posibles dificultades: El antirrebote del pulsador puede ser tricky. Explica que los botones mecánicos rebotan y causan múltiples lecturas. Un pequeño delay o una librería resuelve.

Para el proyecto parcial: Guía a Cargando... a dividir el problema: primero probar cada sensor por separado, luego combinarlos.
