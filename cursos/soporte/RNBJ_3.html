DÍA 7 · 2.5 hrs Introducción a I2C y Sensor AHT20+BMP280
⏱️ 2.5 horas (receso incluido)
Teoría: Protocolo I2C20 min
ComunicaciónProtocolos
I2C usa solo dos cables: SDA (datos) y SCL (reloj). Cada dispositivo tiene una dirección única. Con estos dos cables puedes conectar muchos sensores. Es como el bus de la ciudad: muchos pasajeros, una sola ruta.
⚡ Simulador I2C
💡 El ESP32 tiene pines I2C por defecto: SDA GPIO21, SCL GPIO22.
Presentación del sensor AHT20+BMP28020 min
Este pequeño sensor te da temperatura, humedad y presión. Usa I2C. Solo 4 pines: VCC, GND, SDA, SCL. Cargando..., es como tener una estación meteorológica en miniatura.
📄 Datasheet
Práctica 10: Conexión y primera lectura50 min
Conecta AHT20+BMP280 al ESP32: VCC a 3.3V, GND a GND, SDA a GPIO21, SCL a GPIO22. Instala las librerías Adafruit AHTX0 y Adafruit BMP280 desde el gestor de librerías.
#include <Wire.h> #include <Adafruit_AHTX0.h> #include <Adafruit_BMP280.h> Adafruit_AHTX0 aht; Adafruit_BMP280 bmp; void setup() { Serial.begin(115200); Wire.begin(); if (!aht.begin()) Serial.println("Error AHT20"); if (!bmp.begin()) Serial.println("Error BMP280"); } void loop() { sensors_event_t humidity, temp; aht.getEvent(&humidity, &temp); Serial.print("Temp: "); Serial.print(temp.temperature); Serial.print(" °C, Hum: "); Serial.print(humidity.relative_humidity); Serial.print(" %, Pres: "); Serial.println(bmp.readPressure() / 100.0F); delay(2000); }
❓ ¿No encuentra el sensor? Revisa cables, dirección I2C, y que las librerías estén instaladas.

DÍA 8 · 2.5 hrs Display de 7 Segmentos
⏱️ 2.5 hrs
Teoría: Display de 7 segmentos20 min
Un display de 7 segmentos tiene 8 LEDs (7 para el número, 1 para el punto). Puede ser de cátodo común (todos los negativos juntos) o ánodo común. Para encender un número, debes prender la combinación correcta de segmentos.
⚡ Simulador 7 segmentos
Práctica 11: Conexión y prueba individual55 min
Conecta cada segmento (a, b, c, d, e, f, g) a pines del ESP32 con resistencias de 220Ω. Programa para encender cada segmento uno por uno y entender cuál es cuál.
// Ejemplo para cátodo común (enciende con HIGH) int segA = 13, segB = 12, segC = 14, segD = 27, segE = 26, segF = 25, segG = 33; void setup() { pinMode(segA, OUTPUT); pinMode(segB, OUTPUT); // ... todos } void loop() { digitalWrite(segA, HIGH); delay(500); digitalWrite(segA, LOW); delay(500); digitalWrite(segB, HIGH); delay(500); digitalWrite(segB, LOW); delay(500); // ... y así con todos }
💡 Para ánodo común, enciendes con LOW. Revisa la hoja de datos de tu display.

Práctica 12: Contador con display40 min
Usando SevSeg, programa un contador de 0 a 9 que se incremente cada segundo y se muestre en el display.
#include "SevSeg.h" SevSeg sevseg; void setup() { byte numDigits = 1; byte digitPins[] = {}; // sin digitos múltiples byte segmentPins[] = {13,12,14,27,26,25,33}; // orden: a,b,c,d,e,f,g bool resistorsOnSegments = true; bool updateWithDelays = false; byte hardwareConfig = COMMON_CATHODE; // o COMMON_ANODE sevseg.begin(hardwareConfig, numDigits, digitPins, segmentPins, resistorsOnSegments); sevseg.setBrightness(90); } int contador = 0; void loop() { sevseg.setNumber(contador, 0); sevseg.refreshDisplay(); contador++; if (contador > 9) contador = 0; delay(1000); }

DÍA 9 · 2.5 hrs Pantalla OLED I2C
⏱️ 2.5 hrs
Repaso y tarea15 min
¿Qué ventajas tiene una pantalla OLED sobre un display de 7 segmentos? Puede mostrar texto, gráficos, números grandes, todo en una pantallita de bajo consumo.
Teoría: Pantallas OLED20 min
Usan I2C (mismo SDA/SCL). Resolución común: 128x32 o 128x64. Dirección típica 0x3C. Con librerías de Adafruit dibujarás texto y figuras fácilmente.
📘 Guía OLED
Práctica 13: Hola Mundo en OLED55 min
Conecta OLED: VCC a 3.3V, GND, SDA a GPIO21, SCL a GPIO22. Instala librerías Adafruit SSD1306 y Adafruit GFX.
#include <Wire.h> #include <Adafruit_GFX.h> #include <Adafruit_SSD1306.h> #define SCREEN_WIDTH 128 #define SCREEN_HEIGHT 64 Adafruit_SSD1306 display(SCREEN_WIDTH, SCREEN_HEIGHT, &Wire, -1); void setup() { if(!display.begin(SSD1306_SWITCHCAPVCC, 0x3C)) { Serial.println("No se encuentra OLED"); while(1); } display.clearDisplay(); display.setTextSize(2); display.setTextColor(WHITE); display.setCursor(10, 20); display.println("Hola Bienvenido!"); display.display(); } void loop() {}

Práctica 14: Datos dinámicos en OLED35 min
Combina el potenciómetro (o el sensor AHT20) y muestra su valor en la OLED en tiempo real. Cada medio segundo se actualiza.
#include <Wire.h> #include <Adafruit_SSD1306.h> #include <Adafruit_GFX.h> Adafruit_SSD1306 display(128, 64, &Wire, -1); int potPin = 35; void setup() { display.begin(SSD1306_SWITCHCAPVCC, 0x3C); display.clearDisplay(); display.setTextSize(2); display.setTextColor(WHITE); } void loop() { int valor = analogRead(potPin); display.clearDisplay(); display.setCursor(10, 20); display.print("Valor:"); display.println(valor); display.display(); delay(500); }
💡 Puedes mostrar también temperatura si conectas el sensor AHT20.

❓ Preguntas que se hizo Cargando... esta semana

¿Por qué no funciona el sensor AHT20? Verifica que las librerías estén instaladas y que las conexiones SDA/SCL sean correctas. A veces hay que soldar pines si el sensor viene sin cabezal.
Mi display 7 segmentos no muestra bien los números Revisa la configuración de ánodo/cátodo común y el orden de los pines en la librería SevSeg.
La OLED no muestra nada Verifica dirección I2C (puede ser 0x3C o 0x3D). Ejecuta un scanner I2C para confirmar.
